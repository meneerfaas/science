In Python herken je een functie aan haakjes achter de naam. Bijvoorbeeld \texttt{print()} en \texttt{turtle.penup()} zijn functies. Aan sommige functies kun je informatie meegeven waarmee de functie iets moet doen. Die informatie zet je dan tussen de haakjes. Bijvoorbeeld \texttt{print('Hello, World!')} geeft aan dat de functie \texttt{print()} de tekst \texttt{'Hello, World!'} moet afdrukken.

In dit hoofdstuk ga je je eigen functies definiëren. Dat betekent dat je zelf een naam bedenkt voor een functie en vervolgens Python vertelt wat die functie moet doen wanneer je hem aanroept. Je zult zien dat functies heel handig zijn om je code overzichtelijk te houden. Het zijn eigenlijk de bouwstenen van je programma.

\section{Functies definiëren}%
\label{sec:functies-definieren}
In Python definieer je een functie met het keyword \texttt{def}, gevolgd door de naam van de functie, haakjes en een dubbele punt. Daarna komt de code die bij de functie hoort, de zogenoemde \emph{function body}. In listing~\ref{lst:turtle-functions} zie je een voorbeeld van een functie genaamd \texttt{square()} die een vierkant tekent met de turtle.

\begin{python}[style=mu, caption={\file{turtle\_functions.py}}, label={lst:turtle-functions},name=functionbody, framextopmargin=3pt, framexleftmargin=3pt]
import turtle

tony = turtle.Turtle()

# Functie square() tekent een vierkant
def square():
    for zijde in range(4):
        tony.fd(40)
        tony.lt(90)
\end{python}

 Merk op dat de function body is ingesprongen ten opzichte van de regel waarop de functie is gedefinieerd. Dat is belangrijk, want Python gebruikt inspringing om aan te geven welke code bij de functie hoort. De function body van \texttt{square()} bestaat uit een \texttt{for} loop die een vierkant met zijden van lengte 40 tekent. De regels 8 en 9 zijn ingesprongen ten opzichte van regel 7, zodat Python snapt dat ze bij de \texttt{for} loop horen.

 Wanneer je de code in listing~\ref{lst:turtle-functions} runt, zul je zien dat er niets gebeurt! Daar is een eenvoudige verklaring voor: we hebben de functie \texttt{square()} wel gedefinieerd, maar nog niet aangeroepen. Gelukkig is dat eenvoudig op te lossen.

\begin{python}[style=mu, caption={Functieaanroep in het hoofdprogramma}, label={lst:main-program}]
import turtle

tony = turtle.Turtle()

# Functie square() tekent een vierkant
def square():
    for zijde in range(4):
        tony.fd(40)
        tony.lt(90)

# HOOFDPROGRAMMA
square()
\end{python} 
\clearpage
Op regel 11 in listing~\ref{lst:main-program} staat het commentaar \texttt{\# HOOFDPROGRAMMA}. Dit is een letterlijke vertaling van het Engelse \emph{main program}. In veel programmeertalen is er een speciaal gedeelte van de code dat \emph{main} heet. Het is het deel van de code dat wordt uitgevoerd wanneer het script wordt gestart. Hier roepen we onze functies aan en schrijven we de logica van ons programma. Eigenlijk is dit het startpunt van het programma. De aanroep van \texttt{square()} op regel 12 zorgt ervoor dat de code in de function body van \texttt{square()} daadwerkelijk wordt uitgevoerd.

\begin{infobox}[Main program onderaan]
    Het is gebruikelijk om de code van je hoofdprogramma helemaal onderaan je script te plaatsen, na alle functiedefinities. Op die manier zijn alle functies al gedefinieerd voordat ze worden aangeroepen in het hoofdprogramma. De volgende code zal namelijk een foutmelding geven:

    \begin{python}[style=mu, caption={Verkeerde volgorde}, label={lst:functie-definitie-03}, xleftmargin=.5cm]
# HOOFDPROGRAMMA
my_first_function()

# Mijn eerste functie
def my_first_function():
    print('Groetjes vanuit mijn eerste functie!')
    \end{python}

In regel 2 van  listing \ref{lst:functie-definitie-03} wordt \texttt{my\_first\_function()} aangeroepen, maar deze functie wordt pas gedefinieerd in regel 5. Dit heeft een \texttt{NameError} tot gevolg, omdat Python in regel 2 \texttt{my\_first\_function} nog niet kent.\par
\vspace{8pt}
Vanaf nu is het dus niet meer zo dat de uitvoering van je programma altijd van boven naar beneden verloopt; sterker nog, meestal begint de uitvoering helemaal onderaan, in het hoofdprogramma!
\end{infobox}

\section{Argumenten en parameters}%
\label{sec:argumenten-en-parameters}

Aan de functie \texttt{square()} kun je geen informatie tussen de haakjes meegeven. Maar zou het niet handig zijn als we \texttt{square(100)} konden gebruiken om een vierkant met zijden van 100 pixels te tekenen en \texttt{square(200)} voor zijden van 200 pixels? Ja dat zou heel handig zijn! We willen dus graag informatie kunnen meegeven aan onze functies. In Python noem je die informatie \emph{argumenten}. We gaan de functie texttt{square()} aanpassen zodat het mogelijk wordt om één argument mee te geven: de zijdelengte.

\begin{python}[style=mu, caption={Functie \texttt{square()} met parameter \texttt{zijdelengte}}, label={lst:square-parameter}]
import turtle

tony = turtle.Turtle()

# Functie square() tekent een vierkant
def square(zijdelengte):
    for zijde in range(4):
        tony.fd(zijdelengte)
        tony.lt(90)

# HOOFDPROGRAMMA
square(100)
\end{python}
\clearpage
In listing~\ref{lst:square-parameter} zijn drie dingen gewijzigd ten opzichte van listing~\ref{lst:main-program}.
\begin{enumerate}
\item Om ervoor te zorgen dat \texttt{square()} een argument kan ontvangen, is in regel 6 tussen de haakjes de \emph{parameter} \texttt{zijdelengte} toegevoegd. Parameters zijn variabelen waarin bij een functieaanroep de waarden van de meegegeven argumenten worden opgeslagen. In dit geval heet de parameter \texttt{zijdelengte}, maar je had ook een andere naam kunnen kiezen, zoals \texttt{lengte} of \texttt{size}. De naam van de parameter maakt niet uit, zolang je er maar consequent dezelfde naam voor gebruikt in de function body.
\item In regel 8 is \texttt{tony.fd(40)} gewijzigd in \texttt{tony.fd(zijdelengte)}. Wanneer we nu in het hoofdprogramma \texttt{square(100)} aanroepen, zal Python het argument \texttt{100} toewijzen aan de parameter \texttt{zijdelengte}, zodat er een vierkant met zijden van lengte 100 wordt getekend.
\item In regel 12 is de aanroep van \texttt{square()} gewijzigd in \texttt{square(100)} teneinde een vierkant met zijden van lengte 100 te tekenen.
\end{enumerate}

Met deze aanpassing kunnen we nu vierkanten tekenen van verschillende grootte, door verschillende argumenten mee te geven aan \texttt{square()}, bijvoorbeeld zoals met het hoofdprogramma in listing~\ref{lst:square-different-sizes}.

tony.pd()

\end{python}

Onze \texttt{square()} functie is nu een stuk flexibeler dan voorheen, maar heeft nog wel een nadeel: we kunnen alleen schildpad \texttt{tony} gebruiken en geen andere schildpadden. Stel dat je een programma hebt met twee schildpadden \texttt{tony} en \texttt{tina}, dan heeft \texttt{tina} niks aan de huidige \texttt{square()} functie. Daarom gaan we nog een parameter toevoegen aan \texttt{square()}, zodat we ook de schildpad kunnen specificeren die het vierkant moet tekenen. In listing~\ref{lst:square-turtle-parameter} zie je hoe dat eruitziet.

\begin{python}[style=mu, caption={Functie \texttt{square()} met parameters \texttt{trtl} en \texttt{zijdelengte}}, label={lst:square-turtle-parameter}]
import turtle

tony = turtle.Turtle()
tina = turtle.Turtle()

# Functie square() tekent een vierkant
def square(trtl, zijdelengte):
    for zijde in range(4):
        trtl.fd(zijdelengte)
        trtl.lt(90)

# HOOFDPROGRAMMA
square(tony, 100)
tina.pencolor('red')
tina.lt(45)
square(tina, 141)
\end{python}

Vergelijk de code in listing~\ref{lst:square-turtle-parameter} met de code in listing~\ref{lst:square-parameter}. Je ziet dat een parameter \texttt{trtl} is toegevoegd aan de functie \texttt{square()} en dat deze parameter wordt gebruikt in de function body om te verwijzen naar de schildpad die het vierkant moet tekenen. De volgorde van de parameters in de functiedefinitie mag je zelf kiezen. In listing~\ref{lst:square-turtle-parameter} is de parameter \texttt{trtl} als eerste parameter gedefinieerd en \texttt{zijdelengte} als tweede, maar je had ook kunnen kiezen voor de omgekeerde volgorde. Let er wel op dat je bij het aanroepen van een functie de argumenten in dezelfde volgorde meegeeft als de parameters in de functiedefinitie.

We hebben nu een functie \texttt{square()} die vierkanten tekent met een bepaalde afmeting, penkleur en vulkleur. Maar wat als je nu een vierkant zonder vulling wilt tekenen? \begin{enumerate}[left=0pt .. 2em, itemsep=0.5\baselineskip]
\item Definieer onder de functie \texttt{square()} een nieuwe functie \texttt{filled\_square()} met de parameters \texttt{trtl}, \texttt{size}, \texttt{pnclr} en \texttt{fllclr}.
\item Verwijder uit functiedefinitie van \texttt{square()} de parameter \texttt{fllclr} en verwijder uit de function body van \texttt{square()} alle code die te maken heeft met het instellen van de vulkleur en het vullen van het vierkant. De functie \texttt{square()} moet alleen nog maar een vierkant tekenen met de opgegeven penkleur, maar zonder vulling.
\item Zorg dat de function body van \texttt{filled\_square()} overeenkomt met de code hieronder.
\begin{python}[style=mu, numbers=none, aboveskip=.5\baselineskip]
def filled_square(trtl, size, pnclr, fllclr):
    trtl.fillcolor(fllclr)
    trtl.begin_fill()
    square(trtl, size, pnclr)
    trtl.end_fill()
\end{python}
Zie je dat de functie \texttt{filled\_square()} gebruik maakt van de functie \texttt{square()}? Dat is een groot voordeel van functies: we hoeven niet nogmaals de code te schrijven die een vierkant tekent, maar kunnen gewoon de functie \texttt{square()} aanroepen binnen de function body van \texttt{filled\_square()}.

\end{enumerate}

\section{Coördinaten}%
\label{sec:coordinaten}

Tot nu toe bewoog je de turtle met de functies \texttt{turtle.fd()} en \texttt{turtle.bk()} een aantal pixels vooruit of achteruit. En met \texttt{turtle.lt()} en \texttt{turtle.rt()} kon je de turtle naar links of naar rechts laten draaien om vervolgens een andere richting in te slaan. Maar soms is het handiger de turtle in één keer naar een bepaalde plek in het venster te sturen. Bijvoorbeeld naar de hoek linksonder, of naar het middelpunt. Dat kun je doen door coördinaten te gebruiken.

Coördinaten zijn getallen die aangeven waar een punt ligt in een vlak. Wellicht ben je al bekend met geografische coördinaten die we gebruiken om plaatsen op de wereldkaart aan te duiden. Op Google Maps kun je bijvoorbeeld de coördinaten van Gymnasium Beekvliet in Sint-Michielsgestel vinden: (51.642174818007355, 5.3640463837966434).

Om met coördinaten te werken, heb je een assenstelsel nodig. Meestal gebruiken we een horizontale \(x\)-as en een verticale \(y\)-as. In figuur~\ref{fig:coordinatenstelsel} zie je zo'n assenstelsel met drie punten erin uitgezet: (2, 3), (-4, -1) en (5, -2). Het punt (2, 3) ligt bijvoorbeeld 2 eenheden naar rechts op de \(x\)-as en 3 eenheden omhoog op de \(y\)-as. De oorsprong \(O\) is het punt (0, 0), waar de \(x\)-as en \(y\)-as elkaar kruisen.

\section{De module \texttt{random}}%
\label{sec:random}
Om met turtle te kunnen werken, moest je bovenaan je programma telkens de regel \texttt{import turtle} typen. Daarmee vertel je Python dat je gebruik wilt maken van de functies die in de turtle module zijn gedefinieerd. Er zijn nog veel meer modules beschikbaar in Python, bijvoorbeeld de module \texttt{random} waarmee je willekeurige getallen kunt genereren. Probeer maar eens het programma in listing~\ref{lst:random-walk} uit.

\begin{python}[style=mu, caption={random\_walk.py}, label={lst:random-walk}]
import turtle, random

tony = turtle.Turtle()

for i in range(50):
    x = random.randint(-300, 300)
    y = random.randint(-200, 200)
    tony.goto(x, y)    
\end{python}

In regel 1 van listing~\ref{lst:random-walk} importeren we zowel de \texttt{turtle} module als de \texttt{random} module. In regel 5 en 6 gebruiken we de functie \texttt{randint()} uit de random module om willekeurige \(x\)- en \(y\)-coördinaten te genereren. De argumenten voor \texttt{randint} zijn de ondergrens en bovengrens van het interval waarbinnen het getal gegenereerd wordt. Dus \texttt{random.randint(-300, 300)} genereert een willekeurig geheel getal (een \emph{integer}) tussen -300 en 300. Door dit in een loop te doen en de turtle telkens naar de willekeurig gegenereerde coördinaten te sturen, ontstaat een soort van willekeurige wandeling (zie figuur~\ref{fig:turtle-random-walk}).

for i in range(50):

De kleur rood wordt bijvoorbeeld aangeduid met het woord \texttt{'red'}. Om een willekeurige kleur te kiezen, kun je een lijst maken van alle kleuren die je mooi vindt en in de loop telkens een willekeurig element uit die lijst kiezen met de functie \texttt{random.choice()}. In listing~\ref{lst:random-walk-color} zie je hoe dat er uitziet.

In regel 4 van listing~\ref{lst:random-walk-color} is een lijst \texttt{colors} gedefinieerd met vijf kleuren. Zoals je ziet, moet je in Python vierkante haken (ook wel blokhaken genoemd) gebruiken om een lijst te maken. Met de aanroep \texttt{random.choice(colors)} wordt in regel 10 een willekeurige kleur uit die lijst gekozen en toegewezen aan de variabele \texttt{pc}. Vervolgens wordt in regel 12 deze kleur ingesteld als penkleur met \texttt{tony.pencolor(pc)}.

import turtle, random

tony = turtle.Turtle()
colors = ['red', 'orange', 'green', 'blue', 'purple']

for i in range(50):
    x = random.randint(-300, 300)
    y = random.randint(-200, 200)
    ps = random.randint(3, 8)
    pc = random.choice(colors)
    tony.pensize(ps)
    tony.pencolor(pc)
    tony.goto(x, y)    
\end{python}

import turtle
import random

tony = turtle.Turtle()
tony.screen.bgcolor('dark sea green')
tony.hideturtle()
tony.speed(0)

def flower(trtl, size, petals, petalcolor, centercolor):
    # Bewaar de startpositie voor later
    xstart = trtl.xcor()
    ystart = trtl.ycor()

    # Geef de turtle een willekeurige richting 
    trtl.setheading(random.randint(0, 359))

    # Teken de bloemblaadjes (petals)
    fd_pu(trtl, size * 0.75)
    trtl.lt(90)
    for n in range(petals):
        # Teken eerst een zwarte stip
        trtl.dot(size, 'black')
        # Teken een iets kleinere stip in de petalcolor
        trtl.dot(size - 2, petalcolor)
        # Verplaats een stukje over een cirkelboog
        trtl.pu()
        trtl.circle(size * 0.75, 360 / petals)
        trtl.pd()
    # Verplaats naar de oorspronkelijke positie
    teleport(trtl, xstart, ystart)
    # Teken het hart van de bloem
    trtl.dot(size, centercolor)
    
def teleport(trtl, x, y):
    # Verplaats de turtle zonder te tekenen
    trtl.penup()
    trtl.goto(x, y)
    trtl.pendown()
    
def fd_pu(trtl, afstand):
    # Verplaats de turtle vooruit zonder te tekenen
    trtl.penup()
    trtl.fd(afstand)
    trtl.pendown()
